The **Ocean View Resort Hotel Management System** is a distributed, full-stack web application that digitalises every aspect of modern hotel operations. It provides a **public-facing website** for hotel visibility and **two completely separate authenticated portals** — one for guests and one for staff — with role-based access control across four user types.

> 🎓 This project was developed as a **full-stack IT academic project** demonstrating proficiency in Java EE, database design, MVC architecture, design patterns, and software engineering principles.

## 🏗️ System Architecture

### Three-Tier MVC Architecture

```
  BROWSER / CLIENT
       │  HTTP Request
       ▼
┌─────────────────────────────────────────────────────────────────────┐
│  TIER 1: PRESENTATION LAYER                                         │
│                                                                     │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐               │
│  │  JSP Pages   │  │  HTML/CSS    │  │  JavaScript  │               │
│  │  (Views)     │  │  (Styling)   │  │  (Behaviour) │               │
│  └──────────────┘  └──────────────┘  └──────────────┘               │
└──────────────────────────────┬──────────────────────────────────────┘
                               │ forward / redirect
┌──────────────────────────────▼──────────────────────────────────────┐
│  TIER 2: BUSINESS LOGIC LAYER                                       │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                   CONTROLLER (Servlets)                     │    │
│  │  LoginServlet · BookingServlet · BillServlet                │    │
│  │  CleaningServlet · AdminServlet · ReportServlet             │    │
│  └──────────────────────────┬───────────────────────────────── ┘    │
│                             │  calls                                │
│  ┌──────────────────────────▼──────────────────────────────── ─┐    │
│  │                     MODEL (Java Classes)                    │    │
│  │  Guest · Staff · Room · Reservation · Bill · CleaningReq    │    │
│  │  RoomDAO · ReservationDAO · GuestDAO · BillDAO              │    │
│  └────────────────────────── ┬──────────────────────────────── ┘    │
└──────────────────────────────┼──────────────────────────────────────┘
                               │ JDBC SQL
┌──────────────────────────────▼──────────────────────────────────────┐
│  TIER 3: DATA LAYER                                                 │
│                                                                     │
│  ┌────────────────────────────────────────────────────────────────┐ │
│  │  MySQL Database                                                │ │
│  │  guests · rooms · reservations · cleaning_requests             │ │
│  │  bills · staff · (+ stored procedures + triggers)              │ │
│  └────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### MVC Request Flow

```
  User Action (Click / Form Submit)
          │
          ▼
  HTTP POST/GET to Servlet (Controller)
          │
          ├──► Validate Input (ValidationUtils)
          │
          ├──► Call DAO / Service (Model)
          │         │
          │         └──► SQL Query → MySQL
          │                  │
          │              ResultSet
          │                  │
          │         Return Java Object
          │
          └──► Set Request Attributes
                    │
                    ▼
          forward to JSP (View)
                    │
                    ▼
          HTML rendered to Browser ✅
```

### 🔧 Advanced Database Features

```sql
-- ⚡ STORED PROCEDURE: Monthly Financial Report
DELIMITER //
CREATE PROCEDURE sp_GetMonthlyReport(IN p_year INT, IN p_month INT)
BEGIN
    SELECT
        SUM(b.total_amount) AS total_income,
        COUNT(r.reservation_id) AS total_bookings,
        SUM(b.total_amount) * 0.30 AS estimated_expenses,
        SUM(b.total_amount) * 0.70 AS net_income
    FROM reservations r
    JOIN bills b ON r.reservation_id = b.reservation_id
    WHERE YEAR(r.check_in) = p_year AND MONTH(r.check_in) = p_month
    AND r.status = 'completed';
END //
DELIMITER ;

-- ⚡ TRIGGER: Auto-update room status on booking
CREATE TRIGGER trg_UpdateRoomStatus
AFTER INSERT ON reservations
FOR EACH ROW
    UPDATE rooms SET status = 'occupied'
    WHERE room_id = NEW.room_id AND NEW.check_in = CURDATE();

-- ⚡ FUNCTION: Calculate stay duration
CREATE FUNCTION fn_CalculateStayDuration(checkIn DATE, checkOut DATE)
RETURNS INT DETERMINISTIC
RETURN DATEDIFF(checkOut, checkIn);
```

## ⚙️ Setup & Installation

### Prerequisites

```
✅  Java JDK 17 or higher
✅  Apache Tomcat 10.x
✅  MySQL 8.0 or higher
✅  Maven 3.8+
✅  Any IDE: IntelliJ IDEA / Eclipse / VS Code
✅  Git
```

### Step-by-Step Setup Guide

#### Step 1 — Clone the Repository

```bash
# Clone via HTTPS
git clone https://github.com/yourusername/ocean-view-resort.git

# OR via SSH
git clone git@github.com:yourusername/ocean-view-resort.git

# Navigate into the project
cd ocean-view-resort
```

#### Step 2 — Database Setup

```bash
# Login to MySQL
mysql -u root -p

# Run the setup scripts in order:
mysql -u root -p < database/schema.sql
mysql -u root -p < database/seed_data.sql
mysql -u root -p < database/stored_procedures.sql
```

After running the scripts, verify:

```sql
USE ocean_view_resort;
SHOW TABLES;
-- Expected output:
-- bills
-- cleaning_requests
-- guests
-- reservations
-- rooms
-- staff

SELECT username, role FROM staff;
-- Expected:
-- admin        | ADMIN
-- manager      | MANAGER
-- receptionist | RECEPTIONIST
```

#### Step 3 — Configure Database Connection

Edit `src/main/java/com/oceanviewresort/util/DatabaseConnectionManager.java`:

```java
// 🔧 UPDATE THESE VALUES TO MATCH YOUR MYSQL SETUP
private static final String DB_URL      = "jdbc:mysql://localhost:3306/ocean_view_resort";
private static final String DB_USER     = "root";        // ← your MySQL username
private static final String DB_PASSWORD = "yourpassword"; // ← your MySQL password
```

#### Step 4 — Build the Project

```bash
# Build with Maven
mvn clean install

# This generates: target/oceanviewresort.war
```

#### Step 5 — Deploy to Tomcat

```bash
# Option A: Copy WAR to Tomcat webapps folder
cp target/oceanviewresort.war /path/to/tomcat/webapps/

# Option B: Start Tomcat (if not already running)
cd /path/to/tomcat
./bin/startup.sh        # Linux/Mac
bin\startup.bat         # Windows
```

#### Step 6 — Access the Application 🎉

```
🌐 Public Website:      http://localhost:8080/oceanviewresort/
👤 Guest Login:         http://localhost:8080/oceanviewresort/guestLogin.jsp
👔 Staff Login:         http://localhost:8080/oceanviewresort/staffLogin.jsp
```

## 🚀 CI/CD Pipeline & Deployment

### GitHub Actions Workflow

```yaml
# .github/workflows/ci-cd.yml
name: 🌊 Ocean View Resort CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  # ─── JOB 1: BUILD & TEST ─────────────────────────────────────
  build-and-test:
    name: 🔨 Build & Run Tests
    runs-on: ubuntu-latest

    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: testpassword
          MYSQL_DATABASE: ocean_view_resort_test
        ports:
          - 3306:3306
        options: --health-cmd="mysqladmin ping" --health-interval=10s

    steps:
      - name: 📥 Checkout code
        uses: actions/checkout@v3

      - name: ☕ Set up Java 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: 🗄️ Setup Test Database
        run: |
          mysql -h 127.0.0.1 -u root -ptestpassword < database/schema.sql
          mysql -h 127.0.0.1 -u root -ptestpassword < database/seed_data.sql

      - name: 📦 Cache Maven packages
        uses: actions/cache@v3
        with:
          path: ~/.m2
          key: ${{ runner.os }}-m2-${{ hashFiles('**/pom.xml') }}

      - name: 🏗️ Build with Maven
        run: mvn clean compile

      - name: 🧪 Run JUnit Tests
        run: mvn test
        env:
          DB_URL: jdbc:mysql://127.0.0.1:3306/ocean_view_resort_test
          DB_USER: root
          DB_PASS: testpassword

      - name: 📊 Upload Test Results
        uses: actions/upload-artifact@v3
        if: always()
        with:
          name: test-results
          path: target/surefire-reports/

  # ─── JOB 2: PACKAGE ──────────────────────────────────────────
  package:
    name: 📦 Package WAR
    needs: build-and-test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: 📦 Package WAR file
        run: mvn package -DskipTests

      - name: 📤 Upload WAR artifact
        uses: actions/upload-artifact@v3
        with:
          name: oceanviewresort-war
          path: target/oceanviewresort.war

  # ─── JOB 3: DEPLOY ───────────────────────────────────────────
  deploy:
    name: 🚀 Deploy to Server
    needs: package
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    environment: production

    steps:
      - name: 📥 Download WAR artifact
        uses: actions/download-artifact@v3
        with:
          name: oceanviewresort-war

      - name: 🚀 Deploy to Tomcat via SCP
        uses: appleboy/scp-action@v0.1.4
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          source: "oceanviewresort.war"
          target: "/opt/tomcat/webapps/"

      - name: ♻️ Restart Tomcat
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          script: |
            sudo systemctl restart tomcat
            echo "✅ Deployment complete! Version: ${{ github.sha }}"
```

## 🔒 Security Implementation

```
🛡️  SECURITY FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  🔐 Password Hashing
      All passwords are hashed using BCrypt (work factor 12)
      Plain text passwords are NEVER stored in the database

  🛑 SQL Injection Prevention
      All DB queries use PreparedStatement with parameterised inputs
      User input never concatenated into SQL strings

  🔑 Session Management
      HttpSession created on login, destroyed on logout
      Session timeout: 30 minutes of inactivity
      All protected pages validate session before rendering

  🚫 Authorisation Checks
      Each Servlet checks session role before processing
      Guests cannot access staff endpoints (403 Forbidden)
      Guests can only access their own reservation/bill data

  ✅ Input Validation
      Client-side: JavaScript validation for immediate feedback
      Server-side: Java validation for security enforcement
      Both layers required — client-side alone is insufficient

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 📐 Design Patterns Used

|Pattern             |Where Applied                         |Purpose                                                   |
|:------------------:|:-------------------------------------|:---------------------------------------------------------|
|**MVC**             |Entire application                    |Separate concerns: Servlet (C), Java POJO/DAO (M), JSP (V)|
|**Singleton**       |`DatabaseConnectionManager`           |One DB connection pool instance across the app            |
|**DAO**             |All `*DAO.java` classes               |Abstract database operations from business logic          |
|**Front Controller**|`LoginServlet` → dashboard routing    |Single entry point for authentication routing             |
|**Factory**         |`RoomFactory`                         |Create correct Room subtype based on type string          |
|**Strategy**        |`BillingStrategy` interface           |Pluggable billing calculation algorithms                  |
|**Template Method** |Abstract `Room.calculateNightlyRate()`|Define skeleton, let subclasses fill the rate             |
